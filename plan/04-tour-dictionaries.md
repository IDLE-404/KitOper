# 04 — Driver.js туры: Справочники

- `/first-course/teachers` — Преподаватели

## Тур: Преподаватели (`teachers/index.blade.php`)

**Файл тура:** `resources/js/tours/teachers.js`

### Шаги

#### Шаг 1 — Заголовок
- **Элемент:** `h1`, `.page-title`
- **Заголовок:** Справочник преподавателей
- **Описание:** Список всех преподавателей колледжа. Отсюда управляется состав: кто есть в системе, какие предметы ведёт, какой кабинет по умолчанию. Эти данные используются во всём расписании и в Форме 2.

#### Шаг 2 — Форма добавления преподавателя
- **Элемент:** `form[action*="/teachers"] button[type="submit"]` (кнопка «Добавить» в форме вверху страницы)
- **Заголовок:** Добавить преподавателя
- **Описание:** Нажмите кнопку, чтобы открыть форму добавления. Обязательно укажите **ФИО** и **инициалы** (используются в расписании). Кабинет по умолчанию — комната, куда преподаватель идёт если не назначен другой кабинет.

#### Шаг 3 — Строка преподавателя в таблице
- **Элемент:** `tbody tr:first-child`
- **Заголовок:** Строка преподавателя
- **Описание:** В каждой строке: ФИО, инициалы, кабинет по умолчанию. Оранжевый фон и значок — дубликат (два преподавателя с одинаковым именем). Кнопка **«Редактировать»** — изменить данные. Кнопка **«Удалить»** — удалить (только если не задействован в расписании).

#### Шаг 4 — Назначение предметов
- **Элемент:** `.subject-accordion-toggle` (кнопка «Развернуть предметы» рядом со строкой преподавателя)
- **Заголовок:** Предметы преподавателя
- **Описание:** Каждый преподаватель должен иметь список предметов, которые он ведёт. Это нужно для фильтрации: при назначении пары в расписании система покажет только «разрешённых» преподавателей для выбранного предмета. Отметьте галочками нужные предметы по каждому курсу.

#### Шаг 5 — Дубликаты
- **Элемент:** `.duplicate-badge`
- **Заголовок:** Внимание: дубликат
- **Описание:** Оранжевая строка означает, что два или более преподавателей имеют одинаковое имя. Это может вызвать путаницу в расписании. Проверьте инициалы и при необходимости удалите дублирующую запись.

## Тур: Группы (`groups/index.blade.php`)

**Файл тура:** `resources/js/tours/groups.js`

### Шаги

#### Шаг 1 — Заголовок
- **Элемент:** `h1`, `.page-title`
- **Заголовок:** Справочник групп
- **Описание:** Список всех учебных групп колледжа по курсам. Группы — основной объект расписания: каждая группа получает своё расписание, и для каждой строится Форма 2.

#### Шаг 2 — Форма добавления группы
- **Элемент:** `form[action*="/groups"] button[type="submit"].btn-primary` (кнопка «Добавить» в форме создания группы)
- **Заголовок:** Создать группу
- **Описание:** Нажмите, чтобы добавить новую группу. Укажите: **название** (например, «ИС-12»), **курс** (1–4), **тип** (рус/каз), **есть ли подгруппы**. Тип группы определяет, на каком языке отображаются названия предметов.

#### Шаг 3 — Строка группы
- **Элемент:** `tbody tr:first-child`
- **Заголовок:** Строка группы
- **Описание:** Название группы, курс, тип (рус/каз). Значок **«подгруппы»** означает, что группа делится — в расписании и Форме 2 для неё будут отдельные строки подгрупп 1 и 2.

#### Шаг 4 — Кнопка «Завершить учебный год»
- **Элемент:** `form[action*="finish-year"] button[type="submit"]` (кнопка «Завершить учебный год», красная, вверху страницы)
- **Заголовок:** Завершить учебный год
- **Описание:** Нажмите **«Закончить год»** когда группа заканчивает курс и переходит на следующий (или выпускается). Система переведёт группу на следующий курс и обнулит данные расписания за прошлый год.

## Тур: Дисциплины (`subjects/index.blade.php`)

**Файл тура:** `resources/js/tours/subjects.js`

### Шаги

#### Шаг 1 — Заголовок
- **Элемент:** `h1`, `.page-title`
- **Заголовок:** Справочник дисциплин
- **Описание:** Список предметов, которые ведутся для данного курса. Дисциплины используются при составлении расписания и в Форме 2. Для каждого курса — отдельный список.

#### Шаг 2 — Переключатель курса
- **Элемент:** `#courseSelect` или `.course-tabs`
- **Заголовок:** Выбор курса
- **Описание:** Переключите курс чтобы увидеть дисциплины именно этого курса. Один и тот же предмет может быть в разных курсах — это отдельные записи.

#### Шаг 3 — Добавить дисциплину
- **Элемент:** `button[data-bs-target*="createSubject"]` или `.btn-add-subject`
- **Заголовок:** Добавить дисциплину
- **Описание:** Нажмите, чтобы добавить предмет. Поля: **название на русском**, **название на казахском** (используется для казахских групп), **модуль** (тематический блок), **тип группы** (рус/каз/оба).

#### Шаг 4 — Строка дисциплины
- **Элемент:** `tbody tr:first-child`
- **Заголовок:** Строка предмета
- **Описание:** В строке: название на рус/каз, модуль. Кнопка **«Редактировать»** — изменить. Изменение названия предмета сразу отразится в расписании и Форме 2 везде, где он используется.

## Тур: Аудитории (`rooms/index.blade.php`)

**Файл тура:** `resources/js/tours/rooms.js`

### Шаги

#### Шаг 1 — Заголовок
- **Элемент:** `h1`, `.page-title`
- **Заголовок:** Справочник аудиторий
- **Описание:** Список всех учебных кабинетов. Аудитории используются при назначении пар — система проверяет, не занят ли кабинет в то же время другой группой, и предупреждает о конфликте.

#### Шаг 2 — Добавить аудиторию
- **Элемент:** `button[data-bs-target*="createRoom"]` или `.btn-add-room`
- **Заголовок:** Добавить аудиторию
- **Описание:** Нажмите, чтобы добавить кабинет. Укажите: **код** (например, «101», «Спортзал»), **тип** (стандарт / компьютерный / лаборатория), **активен** (отключённые кабинеты не предлагаются при назначении пар).

#### Шаг 3 — Строка аудитории
- **Элемент:** `tbody tr:first-child`
- **Заголовок:** Строка аудитории
- **Описание:** Код, тип, статус. Неактивные аудитории помечены серым — они недоступны для новых назначений, но старые пары с ними остаются.

#### Шаг 4 — Автоподстановка кабинетов
- **Заголовок:** Автоматическая подстановка кабинетов
- **Описание:** На странице просмотра расписания (режим «День») есть кнопка **«Подставить кабинеты на день»**. Она автоматически назначает свободные кабинеты для всех пар дня, используя список кабинетов из этого справочника. После проверки корректности данных здесь функция работает надёжнее.
- **Элемент:** `body` (если нет конкретного элемента на странице — показываем как общую подсказку)
