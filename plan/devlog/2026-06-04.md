### 1. Режим коррекции в Форме 2 — попап вместо inline-редактирования

**Файл:** `oldApp/resources/views/first_course/form_two.blade.php`

Заменили старый подход (три стекленных `<select>` прямо в ячейке) на плавающий попап `#corrPopover`.

**Как работает:**
- Переключатель «Режим коррекции» добавляет класс `.correction-active` всем `.day-cell`
- Клик на ячейку вызывает `corrPopoverOpen(td)` — позиционирует попап рядом с ячейкой
- Попап показывает 4 кнопки-статуса: Нет занятия / Пара прошла / Другой провёл / Я провёл чужую
- При выборе «Другой провёл» или «Я провёл чужую» — появляются dropdown'ы выбора преподавателя и предмета
- `corrPopReposition()` автоматически перемещает попап если он уходит за правый/нижний край экрана
- «Применить» записывает значения в скрытые `<select>`, затем срабатывает стандартный submit формы

**Ключевые статусы:**

| Ключ | Смысл | Что пишется в БД |
|------|-------|------------------|
| `empty` | Пары не было | `status = 'empty'` |
| `normal` | Пара прошла штатно | `status = 'normal'` |
| `replaced` | Другой преподаватель вёл эту пару вместо меня | `status = 'replaced'`, `replacement_teacher_id` |
| `replacement` | Я провёл пару вместо другого (бонус) | `status = 'replacement'`, `subject_id` |

### 3. Умные подсказки замены в модальном окне пары

**Файл:** `oldApp/resources/views/first_course/schedule/index.blade.php`

Когда в модальном окне редактирования пары виден отсутствующий преподаватель — теперь над переключателем «Включить замену» появляется цветная подсказка:

- **Синяя** (тип больничный / командировка / отпуск / без содержания / по приказу): «Выберите заменяющего преподавателя (предмет остаётся)»
- **Жёлтая** (тип отгул): «Выберите другого преподавателя и другой предмет»

Переключатель замены автоматически включается при открытии окна с отсутствующим преподавателем.

**Как реализовано:**
- В кнопке `✏️` добавлены атрибуты `data-absence-type1` и `data-absence-type2`
- `openModal()` читает эти атрибуты и вызывает `applyAbsenceHint()`
- Список типов с заменой предмета берётся из `@json(\App\Support\TeacherAbsenceTypes::TYPES)`

### 4. Drag-and-drop в редакторе недели

**Файл:** `oldApp/resources/views/first_course/schedule/week.blade.php`

Добавили перетаскивание строк пар для изменения их порядка. Работает только внутри одного дня.

**Логика:**
- Каждый `<tr class="pair-row">` получил атрибут `draggable="true"`
- В ячейку номера пары добавлен иконка-ручка `⠿` (`.drag-handle`)
- `getPairSnapshot(day, pair)` — снимает все значения: 12 select'ов (subj/teach/room × a/a-den/b/b-den) + чекбокс подгруппы + состояние видимости строки
- `applyPairSnapshot(day, pair, snap)` — применяет снимок к другой паре
- `swapPairs(day, p1, p2)` — меняет снимки местами + вспышка зелёного фона на обеих строках
- Перетаскивание на скрытую пару (d-none) заблокировано
- Перетаскивание между разными днями заблокировано

**Почему value-swap, не DOM-swap:** DOM-swap ломает `name` атрибуты полей, которые кодируют `[day][pair]`. Value-swap оставляет DOM нетронутым, переставляет только данные.

**Форма 2 не затронута:** её записи привязаны к `teacher_id + subject_id + date`, а не к номеру слота.

## Известные ограничения

- Drag-drop работает только в одном дне (нельзя перетащить пару из понедельника во вторник) — это сознательное ограничение
- В попапе коррекции Форме 2 нет валидации обязательных полей при статусе `replaced`/`replacement` — пользователь может нажать «Применить» без выбора преподавателя
- GLightbox показывает все скриншоты по одному (не галерея группой по разделу) — можно улучшить позже добавив `data-gallery="section-id"`
