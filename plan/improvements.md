# План улучшений KitOper

## Сложность 1 — Лёгкие (< 1 часа каждая)

### ✅ 1. Валидация в попапе коррекции
**Файл:** `resources/views/first_course/form_two.blade.php`  
**Что:** кнопка «Применить» должна быть disabled если:
- статус `replaced` и `#corrPopTeacher` пустой
- статус `replacement` и `#corrPopSubject` пустой  

**Как:** слушать `change` на select'ах и статус-кнопках → `corrPopValidate()` → toggle `disabled` на `#corrPopApply`

---

### ✅ 2. GLightbox галерея по разделам
**Файл:** `resources/views/docs/index.blade.php`  
**Что:** при открытии картинки должны быть стрелки вперёд/назад по всем скриншотам раздела  
**Как:** в JS-инициализации добавить `a.dataset.gallery = section.id` где `section = img.closest('.docs-section')`

---

### ✅ 3. Confirmation перед «Развернуть семестр»
**Файл:** `resources/views/first_course/schedule/week.blade.php`  
**Что:** перехватить submit формы `.semester-expand-form` → показать Bootstrap-модал с предупреждением о перезаписи → подтвердить/отменить  
**Как:** `e.preventDefault()` → модал с деталями (группа, период) → кнопка «Да, развернуть» сабмитит форму

---

## Сложность 2 — Средние (2–4 часа)

### 4. Конфликты в редакторе недели в реальном времени
**Файл:** `resources/views/first_course/schedule/week.blade.php`  
**Что:** сейчас конфликты (преподаватель занят) проверяются только при сабмите формы. Нужно подсвечивать строку сразу при выборе преподавателя  
**Как:** уже есть `.availability-check` и `data-type="teacher"` на select'ах + `freeTeachersUrl`. При `change` на teacher-select → дебаунс 400ms → AJAX `/free-teachers` → если занят → красная рамка на select + tooltip "занят у группы X в паре Y"  
**Важно:** не блокировать сохранение — только визуальное предупреждение (как сейчас на сабмите)

---

### 5. Журнал замен — модал «Замены на неделю»
**Файл:** `resources/views/first_course/schedule/index.blade.php`  
**Что:** кнопка «Замены недели» (рядом с «Анализ расписания») → модал со списком всех активных замен на текущую неделю  
**Backend:** новый маршрут `GET /schedule/replacements-summary?week_start=&course=` → JSON `[{day, pair, group, absent_teacher, replacement_teacher, subject}]`  
**Frontend:** таблица в Bootstrap-модале, группировка по дням, экспорт в Excel (опционально)

---

## Сложность 3 — Тяжёлые (день+)

### 6. Уведомления о конфликтах в реальном времени
**Файлы:** `index.blade.php` + новый endpoint  
**Что:** после сохранения расписания или при открытии страницы — badge/тост с количеством конфликтов на текущей неделе  
**Как:** polling каждые 60 сек на `GET /schedule/conflicts?week_start=&course=` → если `count > 0` → показывать красный badge на кнопке «Анализ» + тост  
**Сложность:** нужен эффективный SQL-запрос (поиск дублей teacher_id + study_day + lesson_number)

---

### 7. Мобильная адаптация расписания
**Файлы:** `public/css/week-schedule/main.css` + `index.blade.php` (schedule)  
**Что:** на телефоне таблица расписания горизонтально скроллится и нечитаема  
**Как:**
- breakpoint `< 768px`: переключить таблицу на card-layout (каждая пара = карточка)
- В режиме «День»: уже более-менее читаемо, нужна мелкая шлифовка
- В режиме «Неделя»: показывать только один день (текущий), switcher между днями  
**Объём:** ~200 строк CSS + JS для mobile-day-switcher
