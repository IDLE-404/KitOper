# Переход на семестровый норматив часов

## Что изменилось

Система переведена с **месячных нормативов** на **семестровые нормативы**. Это позволяет правильно отслеживать остаток часов в течение семестра.

### Раньше (месячные нормативы)
```
Февраль: норматив 60 часов → провели 12 → остаток 48
Март: норматив 60 часов → провели 8 → остаток 52
(Каждый месяц независимый норматив!)
```

### Сейчас (семестровый норматив)
```
Семестр 1 (сентябрь-январь): норматив 60 часов на весь семестр
  - Февраль: начало 60, провели 12 → остаток 48
  - Март: начало 48, провели 8 → остаток 40
  - (Остаток переносится из месяца в месяц!)
```

## Изменения в БД

### Таблица `form_two_normatives` (и аналоги для курсов 2-4)

**Было:**
```sql
group_id, subject_id, teacher_id, month, year, total_hours, hours_per_class
```

**Стало:**
```sql
group_id, subject_id, teacher_id, semester, total_hours, hours_per_class
```

- **semester**: 1 или 2 (вместо month/year)
- **total_hours**: норматив на **весь семестр** (не на месяц!)

### Миграция

При запуске миграции `2027_07_02_000000_migrate_form_two_normatives_to_semester_based`:
- Все старые месячные нормативы **преобразуются** в семестровые
- Для каждого (group_id, subject_id, teacher_id, semester) берётся **максимальное значение** total_hours
- Данные сохраняются, таблица переструктурируется

## Логика расчёта часов

```php
$semester = ($month >= 9 || $month <= 1) ? 1 : 2;

// Семестр 1: сентябрь-январь (месяцы 9, 10, 11, 12, 1)
// Семестр 2: февраль-август (месяцы 2, 3, 4, 5, 6, 7, 8)

// hours_left_start = остаток на НАЧАЛО месяца
// hours_left = остаток на КОНЕЦ месяца
```

## Как это работает в Форме 2

1. **hours_left_start** = total_hours (семестр) - часы ДО этого месяца
   - Февраль: 60 - 0 = 60
   - Март: 60 - 12 = 48
   - Апрель: 60 - 20 = 40

2. **hours_left** = total_hours (семестр) - часы ДО и В этом месяце
   - Февраль: 60 - 12 = 48
   - Март: 60 - 20 = 40
   - Апрель: 60 - 30 = 30
